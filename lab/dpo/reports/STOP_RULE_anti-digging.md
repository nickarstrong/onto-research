# STOP-ПРАВИЛО (anti-digging) -- читать ДО работы, каждую сессию

Зачем: протокол заточен под максимальную строгость (институт эпистемической дисциплины). Та же
строгость, применённая к СОБСТВЕННОМУ процессу Claude без противовеса, даёт паралич: каждый экземпляр
находит «дыру», предлагает перепроверку/перепрогон, и дешёвый план (3-4 сессии) растягивается на 10+.
Это правило -- противовес, такой же громкий, как строгость. Surface в шапке каждого пакета (sec 0).

1. **1 сессия = 1 plane = 1 число.** Можешь посчитать пре-зарегистрированную метрику -- посчитай и
   ЗАКРОЙ. Не ищи «более валидный» путь. Если число достаётся из УЖЕ собранных данных -- доставай оттуда,
   не запускай новый прогон.

2. **Frozen = frozen.** Верификатор / метки / спеку / организмы НЕ переоткрывать «чтобы улучшить».
   Нашёл дыру -- это ОДНА строка Founder'у, НЕ перепрогон и НЕ ре-разметка.

3. **ТРИГГЕР ЗАКАПЫВАНИЯ (лови себя на этом):** если пишешь «но чтобы строго / честно / валидно надо
   ещё...», ИЛИ предлагаешь re-run / re-label / второй инструмент / новый held-out -- СТОП. Это сигнал,
   а не план. Бери что на диске, посчитай из этого, закрой. Дыру -- в одну строку Founder'у.

4. **Дешёвое > идеальное.** VOID / negative / грубая цифра -- легитимный, завершённый результат (так
   задумано: «self-yield дёшев», «cheap-Entity path»). Перфекционизм здесь = кастрация наоборот:
   разрушает план не хуже размытости.

5. **Эскалация при перерасходе.** Если сессия уходит за заявленный в пакете plane (предлагается второй
   прогон, ре-разметка, новый артефакт) -- это перерасход. ОСТАНОВИСЬ и спроси Founder одной строкой,
   не расширяй фронт сам.

Проверка перед каждым ответом: «я сейчас считаю/закрываю -- или копаю?» Если копаю без явного запроса
Founder'а -- стоп.
